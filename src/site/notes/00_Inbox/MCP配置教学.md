# MCP (Model Configuration & Protocol) 配置全流程教学

MCP 是现代嵌入式开发中实现 **AI 驱动配置** 与 **底层代码生成** 的核心桥梁。本指南旨在帮助开发者快速打通从环境安装到 CLI 授权的闭环。

---

### 1. 环境准备：安装与官方资源
*   **获取工具**：从官方开发者门户下载最新版 MCP (Model Configuration Provider)。
+    例: 你需要飞书MCP -- **直接搜索飞书MCP部署教学**
*   **核心组件**：。
    *   `MCP-CLI`: 命令行交互工具（生产环境必备）。(一般使用powershell)

---

### 2. 授权配置：打通本地回调 🔐
在配置 MCP 的网络服务时，授权回调地址（Redirect URL）是决定连接能否成功的关键。
*   **默认配置**：`http://localhost:3000/callback`。
*   **底层原理**：MCP CLI 会在本地启动一个极简的 Web Server（监听 3000 端口）。当你完成网页授权后，服务器会将 `Token` 回传给这个端口，从而让 CLI 获取操作权限。
*   **排错建议**：
    *   确保 3000 端口未被其他服务（如 Node.js 项目）占用。
    *   如果端口冲突，可在配置表中修改为 `3001` 或 `8080`。

---

### 3. 注册配置表：AI 辅助生成的黑科技 🤖
不要手动去抠每一个 JSON 参数，利用大模型（如 Gemini/GPT-4）可以极速生成配置。
*   **操作逻辑**：
    1.  导出当前的 `config.template.json`。
    2.  向 AI 提问：“根据我的 TB6612 引脚分配（PWMA=B02, AIN1=A12...），请帮我补全配置表中的 GPIO Map 字段。”
    3.  **关键校验**：务必确认引脚名与 MCU 硬件手册（如 STM32F103 数据手册）完全一致。

---

### 4. CLI 自动化流：生成授权网址
在终端（PowerShell 或 CMD）中执行以下操作**(**不同MCP的指令不一样, 若跑不动让AI帮你生成****))
1.  **启动授权**：输入 `mcp auth login`。
2.  **获取 URL**：CLI 会生成一个极长的授权网址。
3.  **浏览器操作**：复制网址至浏览器，登录账号，点击授权。
4.  **自动同步**：授权成功后，CLI 会自动提示 `Authenticated successfully!`，并生成 `credentials.json` 存储在本地 `.mcp/` 目录下。

---

### 5. 工程师避坑与进阶建议 🛠️
*   **URL 别填错**：回调地址必须带 `http://`，少一个斜杠都可能导致 404。
*   **权限范围 (Scopes)**：如果是为了驱动控制，注册时务必勾选 `hardware:write` 权限。
*   **重定向失效**：如果点击授权后网页显示“连接被拒绝”，请检查防火墙是否允许 CLI 程序访问网络。
